Dorking

ما هو الـ Dorking؟
استخدام معاملات بحث متقدمة (Google Dorks) لاكتشاف المعلومات المخفية والنقاط الضعيفة بسرعة. 
https://dorks.xen00rw.me/         
https://guilherme-moraiss.github.io/Deep-Dork-Web/         
https://taksec.github.io/google-dorks-bug-bounty/         
https://darkzone964.github.io/dorkme/dorkme.html

Passive Subdomain

لماذا subfinder؟
أداة سريعة مفتوحة المصدر تستقصي مصادر عامة ومجموعات بيانات لاستخراج أسماء النطاقات الفرعية دون إرسال حزم إلى الهدف (سلبي بالكامل). لا تُحدث ضوضاء، تعطي نتائج فورية، وتدعم أكثر من 25 مصدراً مثل VirusTotal، Crt.sh، Cloudflare وغيرها. 
subfinder -d example.com -all -recursive -timeout 20 -o subdomains.txt
-d example.com: النطاق المستهدف.
-all: استخدام جميع المصادر المتاحة.
-recursive: البحث بشكل تكراري لاستخراج دومينات أعمق.
-timeout 20: مهلة 20 ثانية لكل مصدر.
-o subdomains.txt: حفظ النتائج في الملف المحدد. 
subfinder -dL subs.txt -all -recursive -timeout 20 -o subdomains.txt
-dL subs.txt: قائمة من النطاقات (ملف نصي).
بقية الأعلام كما في الأمر السابق.

تحميل الأداة: subfinder

Active Subdomain

لماذا puredns؟
أداة فرعية سريعة تعتمد على DNS مباشرة لاختبار ملايين الاستعلامات في الثانية. تكشف دومينات غير منشورة، تدعم وضع Bruteforce ووضع Resolver سريع، وتُنقي النتائج تلقائياً. 
puredns bruteforce wordlist.txt example.com -r resolvers.txt -w brute_subs.txt
bruteforce: وضع التخمين القاموسي.
wordlist.txt: ملف القاموس (كلمات السب-دومين).
example.com: النطاق الأساسي المستهدف.
-r resolvers.txt: ملف محلِّلات DNS (للدقة والسرعة).
-w brute_subs.txt: اسم الملف الناتج بالنطاقات الصالحة.
لماذا ffuf؟
أداة تخمين سريعة تعتمد على الـ HTTP لاكتشاف الـ Virtual Hosts أو المسارات أو الملفات. تُظهر مواقع فرعية تعمل على نفس IP لكنها تُستدعى عبر رأس Host مختلف، مما يكشف خدمات مخفية. 
ffuf -u http://www.example.com          -H "Host: FUZZ.example.com" -w vhosts.txt -mc 200 -o vhost_results.txt
-u http://www.example.com : الرابط الأساسي الذي سيتم اختباره.
-H "Host: FUZZ.example.com": تعديل رأس Host ليكون FUZZ.example.com.
-w vhosts.txt: قائمة كلمات الـ vhost (ستُستبدل بدل FUZZ).
-mc 200: إظهار النتائج التي ترجع حالة 200 فقط.
-o vhost_results.txt: حفظ النتائج في ملف نصي.
لماذا altdns؟
أداة تُحدث طفرات (mutations) على أسماء النطاقات باستخدام كلمات شائعة. تولِّد دومينات جديدة مثل (dev-api, api-dev, dev-backup) قد تكون غير مدرجة في أي مصدر. 
altdns -i subdomains.txt -o mutated.txt -w /usr/share/seclists/Discovery/DNS/subdomains-top1million-110000.txt
-i subs.txt : مجموعة من النطاقات الاساسية وليست الفرعية.
-o mutated.txt: اسم الملف الناتج بالدومينات المُحورة.
-w /usr/share/seclists/Discovery/DNS/words.txt: قائمة الكلمات المستخدمة للطفرات.
قاموس جاهز (SecLists)
أكبر مشروع قواميس اختبار الاختراق يحتوي على ملايين الكلمات الجاهزة للطفرات والتخمين.
⇢ تحميل قاموس DNS من SecLists

Check Alive

لماذا httpx؟
أداة سريعة مخصصة لاكتشاف الخدمات الحيّة عبر HTTP/HTTPS مع دعم التوجيهات، التحقق من الشهادات، والتصفية حسب الحالة. تُظهر فقط الروابط التي تستجيب بشكل فعلي، تدعم العمل على ملفات ضخمة، وتُخرج النتائج جاهزة للخطوة التالية.
▎ أوامر الفحص الأساسية
cat allsubdomian.txt | httpx-toolkit -silent -o alive.txt
cat allsubdomian.txt: يدخل قائمة الدومينات التي جمعناها سابقاً.
-silent: يُخفي رسائل التقدم ويُظهر فقط الروابط الحية.
-o alive.txt: يحفظ الروابط النشطة في ملف نصي جاهز للفحص أو الاختبار اللاحق. 
httpx-toolkit -l allsubdomin.txt -ports 80,443,8080,8443 -o alive_ports.txt
-l allsubdomian.txt: قائمة الدومينات من ملف.
-ports 80,443,8080,8443: يختبر فقط المنافذ الشائعة (HTTP، HTTPS، وغيرها).
-o alive_ports.txt: يحفظ الروابط النشطة مع رقم المنفذ في ملف منفصل. 
cat allsubdomain.txt | httpx-toolkit -probe -o alive_probe.txt
-probe: يُجري فحصًا نشطًا (TCP connect) على المنفذ الافتراضي 80/443 بدون إرسال طلب HTTP كامل، مما يُسرّع عملية التحقق ويقلّل من الضجيج.
-o alive_probe.txt: يحفظ فقط النطاقات التي استجابت على مستوى الاتصال TCP.
لماذا نستخدمه؟ عندما تكون لديك قائمة ضخمة وتحتاج معرفة «من يستجيب» بأقل استهلاك للوقت والموارد، قبل أن تبدأ فحص HTTP تفصيليًا. 
cat allsubdomain.txt | httpx-toolkit -ip -cdn -o alive_ip.txt
-ip: يعرض عنوان IP الخاص بكل هدف في نفس سطر الإ -cdn: يكشف ويُظهر اسم مزود CDN أو الوكيل (Cloudflare، AWS، إلخ) إن وُجد.
-o alive_ip.txt: يحفظ النتائج في ملف نصي يحتوي على الهدف + IP + CDN.
الفائدة: معرفة أي أهداف تُدار عبر وكيل أو CDN مباشرة من دون حاجة لفحص يدوي، مما يساعدك على ترتيب الأولويات (أهداف مباشرة أولاً).
▎ أدوات أخذ لقطات للصفحات
لماذا نلتقط صوراً للصفحات؟
يسمح لك بمراجعة بصرية سريعة لكل تطبيق حيّ؛ تستطيع اكتشاف صفحات تسجيل الدخول، أخطاء، معلومات تسريب، أو واجهات إدارة ظاهرة فوراً دون فتح كل رابط يدوياً.
لماذا aquatone؟
سريع، يعتمد على Go، يُجرّب عدة منافذ في وقت واحد، ويُخرج صوراً عالية الجودة مع ملف HTML يُعرض فيه جميع اللقطات. 
cat alive.txt | aquatone -out screenshots
cat alive.txt: يمرر الروابط الحية إلى الأداة.
-out screenshots: مجلد يُحفظ فيه جميع الصور وملف HTML النهائي.
النتيجة: ملف screenshots/aquatone_report.html يعرض لك كل اللقطات المصغّرة في صفحة واحدة.
لماذا gowitness؟
يُستخدم Chrome-headless داخليًا ليعطي صوراً دقيقة، ويدعم إضافة أوامر مخصصة (مثل إخفاء الإعلانات أو إضافة ترويسات)، ويُخرج تقريراً HTML مع فلترة حسب حالة الاستجابة. 
gowitness scan file -f live.txt -s screenshots/
file -f alive.txt: يقرأ الروابط من الملف مباشرة.
-s screenshots/: يحدد المسار الذي تُخزن فيه الصور.
النتيجة: ملف gowitness_report.html يحتوي على معاينات مع معلومات الحالة والعناوين.
لماذا eyewitness؟
يدعم أكثر من 10 متصفحات (Chromium، Firefox…) ويُخرج تقارير HTML منظَّمة مع فهرس بحسب حالة الاستجابة (200، 401، 500 …)، ويُمكنه التقاط صفحات تسجيل الدخول تلقائياً. 
eyewitness --web -f alive.txt -d eyewitness_report
--web وضع فحص الويب (URLs).
-f alive.txt: قائمة الأهداف الحية.
-d eyewitness_report: مجلد يُخرج فيه التقرير والصور.
النتيجة: افتح eyewitness_report/report.html لتجد معاينات مرتبة حسب الحالة مع عرض سريع للعنوان والعنوان الوحيد.

Web Content Discover

لماذا نكتشف المحتوى السلبي؟
نبحث عن المسارات والملفات والمعلومات المخفية التي تمّ أرشفتها أو تسريبها في السابق دون إرسال طلبات مباشرة إلى الهدف، مما يقلّل الضوضاء ويزيد احتمالية العثور على نقاط ضعف أو معلومات حسّاسة.
▎ Passive (أرشيف وروابط تاريخية)
لماذا waybackurls؟
أداة Go سريعة تستخرج جميع الروابط التي ظهرت تاريخياً على الهدف من أرشيف Wayback Machine (archive.org)، مما يمنحك قائمة ضخمة من المسارات والملفات القديمة دون إرسال حزمة واحدة إلى الهدف. 
cat allsubdomain.txt | waybackurls > wayback_links.txt
cat allsubdomain.txt: يمرّر جميع النطاقات الفرعية إلى الأداة.
waybackurls: يستعلم عن كل نطاق في أرشيف Wayback Machine ويُخرج الروابط.
> wayback_links.txt: يحفظ الروابط التاريخية في ملف نصي جاهز للفلترة أو الفحص اللاحق.
لماذا gau (GetAllUrls)؟
أداة مشابهة تدمج مصادر متعددة: Wayback Machine، Common Crawl، وـ URLScan، ثم تُخرج قائمة واحدة مُنقّاة من الروابط القديمة. 
cat allsubdomain.txt | gau | tee gau.txt
cat allsubdomain.txt: يمرّر النطاقات.
gau: يجلب الروابط من مصادر الأرشيف المتعددة.
| tee gau.txt: يطبع النتائج أمامك ويحفظها مباشرة في gau.txt.
لماذا waymore؟
تُطوّر من نفس فريق الـ waybackurls، لكنها تُضيف مصادر أكثر (مثل AlienVault وـ URLScan الحديث) وتُخرج روابط مُنقّاة مع إمكانية استخراج ملفات JS أو ترويسات الاستجابة أو حتى أجسام الصفحات إن رغبت. 
waymore -i allsubdomain.txt -mode U -oU links_waymore.txt
-i allsubdomain.txt: ملف يحتوي على النطاقات الفرعية (كل نطاق في سطر).
-mode U: وضع الروابط فقط (URLs) بدون تنزيل أجسام الصفحات.
-o links_waymore.txt: الملف الناتج بالروابط التاريخية.
النتيجة: قائمة شاملة ومُنقّاة جاهزة لفحص سريع أو اختبار تلقائي لاحق.
▎ Active Directory (تخمين مسارات وملفات مباشر)
لماذا dirsearch؟
أداة بايثون شهيرة لاختبار المحتوى تعتمد على قوائم كلمات وتدعم الترويسات المخصصة، والأعمدة العشوائية، والتصفية حسب الحالة، وتُناسب اختبارات الدليل والملفات. 
python3 dirsearch.py -L subs.txt -w wordlist.txt -t 50 --random-agent --header="X-Forwarded-Host: 127.0.0.1" --header="X-Original-URL: /" -o dirsearch_out.txt
-L subs.txt: قائمة الروابط الحية (كل رابط في سطر).
-w wordlist.txt: قاموس المسارات أو الملفات.
-t 50: عدد الخيوط (50 موضوعاً) لتسريع العملية.
--random-agent: يُغيّر User-Agent عشوائياً في كل طلب لتجنب الحظر.
--header="X-Forwarded-Host: 127.0.0.1": يُضيف ترويسة تُخادع بعض أدوات التوجيه.
--header="X-Original-URL: /": ترويسة إضافية قد تُظهر مسارات مخفية.
-o dirsearch_out.txt: يحفظ النتائج في ملف نصي منظم.
لماذا feroxbuster؟
مكتوبة بلغة Rust فهي أسرع، تدعم التخمين المتكرر (recursive)، وتصفية حالات معينة، وترويسات مخصصة، وتُناسب اختبار الملفات والأدلة معاً. 
cat live.txt | feroxbuster -w /usr/share/seclists/Discovery/Web-Content/raft-medium-directories-lowercase.txt -t 50 -x php,html,js,txt -H "User-Agent
: curl/7.68.0" -H "X-Originating-IP: 127.0.0.1"   --stdin -o ferox_out.txt
cat live.txt: يقرأ الروابط من الملف (كل رابط في سطر).
-w wordlist.txt: قاموس المسارات أو الملفات.
-t 50: عدد الخيوط (50) لتسريع العملية.
-x php,html,js,txt: يُلحق هذه الامتدادات بكل مسار (يبحث عن ملفات).
-H "User-Agent: curl/7.68.0": ترويسة User-Agent مخصصة.
-H "X-Originating-IP: 127.0.0.1": ترويسة إضافية قد تُخدع بها بعض أدوات التوجيه.
-o ferox_out.txt: حفظ النتائج في ملف نصي منظم. --stdin أضفت --stdin في النهاية حتى يعرف feroxbuster ياخذ URLs من cat live.txt.
لماذا ffuf في وضع المسارات؟
تُتيح لك استخدام قاموسين في آنٍ واحد (روابط + مسارات) داخل نفس الأمر، مع فلترة حالات الاستجابة والترويسات المخصصة، وبسرعة عالية جداً. 
ffuf -u https://FUZZ/PATH -w subs.txt:FUZZ -w wordlist.txt:PATH -mc 200,204,301,302,403 -fc 404 -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64)" -H "X-Forwarded-For: 127.0.0.1" -t 80 -o ffuf_out.json
-u https://FUZZ/PATH: رابط القالب حيث يُستبدل FUZZ بالنطاق وPATH بالمسار.
-w subs.txt:FUZZ: قاموس النطاقات (متغير FUZZ).
-w wordlist.txt:PATH: قاموس المسارات (متغير PATH).
-mc 200,204,301,302,403: إظهار هذه الحالات فقط.
-fc 404: استبعاد الاستجابات 404 تماماً.
-H "User-Agent: ..." و -H "X-Forwarded-For: 127.0.0.1": ترويسات مخصصة لتجنب الحظر أو كشف المسارات المخفية.
-t 80: عدد الخيوط (80) لتسريع العملية.
-o ffuf_out.json: حفظ النتائج في ملف JSON منظم.
▎ Parameter Discover (العثور على معاملات URL)
لماذا paramspider؟
أداة بايثون خفيفة تبحث عن معاملات URL (?id=& / ?page=&) داخل الملفات النصية أو الروابط الحية، مع إمكانية استبعاد امتدادات معينة والتحكم بعمق البحث؛ مفيدة لاكتشاف نقاط حقن أو مسارات مخفية. 
python3 paramspider.py -l subs.txt
-l subs.txt: قائمة الروابط الحية (كل رابط في سطر).
لماذا fallparams؟
أداة Go سريعة تعتمد على قوائم كلمات مدمجة وتُخرج روابط تحتوي على معاملات ديناميكية، تدعم إدخال ملف روابط أو قراءة من stdin، وتُناسب اختبارات حقن المعاملات أو اكتشاف النقاط غير المنشورة. 
fallparams -u "/path/to/file.txt"
-u "/path/to/file.txt": الملف الذي يحتوي على الروابط (كل رابط في سطر).
النتيجة: تُطبع المعاملات المكتشفة مباشرة في الطرفية ويمكن توجيهها إلى ملف:
fallparams -u urls.txt > params_out.txt

تحميل الأدوات:
paramspider | fallparams

Information Disclosure

ماهو Information Disclosure؟
هي ثغرة أمنية تؤدي إلى تسريب معلومات حساسة أو خاصة بالنظام، مثل:
  • مسارات ملفات داخلية أو بنية المجلدات.
  • مفاتيح API، كلمات سر، أو روابط إدارة مخفية.
  • ملفات إعدادات مثل .env أو config.php.
  • ملفات نسخ احتياطية أو سجلات (logs).
  • تعليقات مطورين تحتوي على بيانات حساسة.
▎ لماذا تُعد خطيرة؟
تُعطي المهاجم فكرة واضحة عن بنية التطبيق، تقنيات المستخدمة، وأحيانًا تُفتح له باب للوصول إلى ثغرات أعمق مثل:
• Local File Inclusion (LFI) أو Remote Code Execution (RCE).

Server Banner Version

ما هو Server Banner؟
هو الحقل Server في رأس الاستجابة HTTP يكشف عن نوع الخادم وإصداره مثل: 
Server: nginx/1.18.0 (Ubuntu)
أو 
Server: Apache/2.4.41 (Win64) OpenSSL/1.1.1c
▎ الفائدة من كشف الإصدار
• معرفة التقنيات المستخدمة يُسرّع عملية التخطيط للهجوم.
• البحث في قواعد بيانات الثغرات (CVE) عن ثغرات معروفة في الإصدار المكتشف.
• تحديد آخر إصدار آمن والضغط من أجل التحديث.
▎ الخطورة
إظهار الإصدار الدقيق = خريطة طريق للمهاجم:
  • ثغرات عامة مثل CVE-2021-41773 لـ Apache 2.4.49.
  • ثغرات خاصة بالإضافات (modules) المثبتة.
  • تخطيط هجمات مخصصة (exploit kits) تعتمد على الإصدار.
httpx-toolkit -tls-grab -title -tech-detect -web-server -status-code -content-length -o http_banners.txt
الأمر الشامل لجمع أكبر قدر من البيانات السطحية في لقطة واحدة:
-tls-grab يسحب شهادة TLS ويعرض CN، SAN، Issuer وتواريخ الانتهاء.
-title يعرض عنوان الصفحة (title tag) مباشرة.
-tech-detect يكشف التقنيات المستخدمة (مثل PHP, ASP.NET, Node.js) عبر التوقيعات.
-web-server يستخرج حقل Server الكامل (Banner) بدقة.
-status-code يضيف رمز الاستجابة HTTP (200, 404 …).
-content-length يضيف طول الجسم (Content-Length) لتمييز الصفحات الديناميكية.
-o http_banners.txt يحفظ كل هذه المعلومات في ملف واحد منظم وسهل الفلترة.
الفائدة: تجميع خارطة تقنية شاملة للأهداف الحية خلال ثوانٍ دون الحاجة لإرسال طلبات يدوية متكررة. 
cat live.txt | while read u; do echo "=== $u ==="; curl -Is --max-time 10 "$u" | sed -n '1,20p'; done > banners_curl.txt
حلقة bash بسيطة لجلب رؤوس الاستجابة الأولى لكل هدف:
while read u يقرأ كل رابط من ملف live.txt سطرًا سطرًا.
echo "=== $u ===" يطبع فاصل بين النتائج لتسهيل القراءة.
curl -Is --max-time 10 "$u" يرسل طلب HEAD فقط مع مهلة 10 ثوانٍ.
sed -n '1,20p' يقتطع أول 20 سطرًا من الرد (يكفي لرؤية الرؤوس المهمة).
> banners_curl.txt يحفظ كل الرؤوس في ملف نصي موحد.
الفائدة: تعمل حتى في البيئات التي لا تدعم httpx-toolkit، وتعطيك رؤية سريعة على البنية العامة للخادم. 
OPENSSL_BIN=$(command -v openssl) && perl -ne 'chomp; next if /^\s*$/; s{^[a-zA-Z]+://}{}; s{/.*$}{}; s{.*@}{}; next if /^(127\.0\.0\.1|localhost)$/; print "$_\n";' live.txt | awk '{ if ($0 !~ /:/) print $0":443"; else print }' | sort -u | xargs -I{} bash -c './testssl.sh --openssl="$1" -U "$2"' _ "$OPENSSL_BIN" {}

Error Messages

ما هي رسائل الخطأ؟
هي الاستجابات التي يُرجعها الخادم عند حدوث مشكلة (مثل 404، 500، 403) وغالبًا ما تحتوي على تفاصيل داخلية مثل:
  • مسارات ملفات حقيقية على القرص.
  • أنواع قواعد البيانات المستخدمة.
  • إصدارات اللغات أو الإطارات (PHP، ASP.NET، Django).
  • متغيرات بيئة أو أسماء مستخدمين داخليين.
▎ الفائدة من جمع رسائل الخطأ
• تكشف بنية المجلدات وتقنيات الخادم دون الحاجة لطلبات استكشاف إضافية.
• تساعد على تقليل نطاق البحث عن الثغرات عن طريق معرفة الإصدار الدقيق.
• تُستخدم لبناء هجمات أكثر دقة (مثل LFI، SQLi، RCE) بناءً على المعلومات المسربة.
▎ الخطورة
إظهار تفاصيل الخطأ = تسريب معلومات حساسة:
  • مسار كامل قد يكشف نظام التشغيل أو أسماء المستخدمين.
  • خطأ قاعدة بيانات قد يكشف نوع الـ DB وحقول الجدول.
  • stack trace قد يعرض دوال داخلية أو مفاتيح API.
cat urls.txt |

Source Code Disclosure

ما هو Source Code Disclosure؟
ثغرة تسمح للمهاجم بالوصول إلى الكود المصدري للتطبيق (PHP، Python، Ruby، ASP.NET…) بدلاً من تنفيذه على الخادم، مما يكشف:
  • مفاتيح API، أسرار، كلمات سر قواعد البيانات.
  • منطق الأعمال والمزايا المخفية أو المسارات الإدارية.
  • ثغرات أمنية واضحة (SQL queries، system calls، deserialization).
  • تعليقات المطورين التي تحتوي على بيانات حساسة.
▎ الخطورة
• كشف الكود يُزيل طبقة الأمان بالكامل ويُعطي المهاجم خارطة الطريق لاستغلال الثغرات.
• المهاجم يستطيع تخطيط هجمات دقيقة (RCE، LFI، SSRF) بناءً على ما قرأه في الكود.
• تسريب الأسرار يؤدي إلى اختراق كامل للبنية التحتية (قواعد البيانات، خدمات السحابة، إلخ). 
while IFS= read -r u; do
  for suf in .bak .old .swp .save ~ .1 .orig .dist _backup .back .txt .zip .tar.gz; do
    wget -q --timeout=4 "${u%/*}/${u##*/}${suf}" -O - | grep -m 1 -iE "(\$|include|password|mysql)" && echo "[!] SRC -> ${u%/*}/${u##*/}${suf}"
  done
done < urls.txt | tee backup-disclosure.txt
فحص أوتوماتيكي للنسخ الاحتياطية والملفات المُحَورة التي قد تُظهر الكود المصدري:
while IFS= read -r u يقرأ كل رابط من urls.txt دون تجزئة المسافات أو الرموز الخاصة.
for suf in … يجرب 12 لاحقة شائعة للنسخ الاحتياطية (.bak، .old، .swp، .save، ~، .1، .orig، .dist، _backup، .back، .txt، .zip، .tar.gz).
wget -q --timeout=4 "${u%/*}/${u##*/}${suf}" يُرسل طلب صامت بمهلة 4 ثوانٍ للملف الاحتياطي المُحَوَّر.
grep -m 1 -iE "(\$|include|password|mysql)" يبحث في المحتوى المُرجع عن مؤشرات كود حقيقي (علامة $، include، password، mysql).
&& echo "[!] SRC -> …" إذا وُجدت إحدى الكلمات يُطبع رابط الملف الذي يحتوي الكود المصدري.
| tee backup-disclosure.txt يحفظ جميع النتائج في ملف نصي لمراجعتها لاحقًا.
الفائدة: خلال ثوانٍ تحصل على قائمة قصيرة بالملفات الاحتياطية التي تسرب الكود المصدري، مما يتيح قراءة الملفات دون استغلال معقد، وسرعة اكتشاف أسرار أو ثغرات مباشرة.

Configuration Files

ما هي ملفات الإعدادات؟
ملفات تحتوي على متغيّرات بيئة التشغيل وأسرار التطبيق مثل:
  • مفاتيح API، توكنات OAuth، أسرار JWT.
  • بيانات الاتصال بقواعد البيانات (host، username، password).
  • إعدادات البريد، خدمات السحابة، وروابط الإدارة.
  • طرق التشفير، مسارات الملفات الحساسة، وقواعد إعادة الكتابة.
▎ الخطورة
• الوصول إلى ملف إعدادات كافٍ للسيطرة الكاملة على التطبيق والبنية التحتية.
• كشف قاعدة البيانات يسمح بالاستخراج الكامل أو التعديل/الحذف دون وجود ثغرة أخرى.
• تسريب المفاتيح يفتح باب الإساءة إلى خدمات الطرف الثالث، الرفع الغير مصرّح به، أو الرسوم المالية.
• حتى إصدار ملف .env أو config.php دون تفعيل وضع التصحيح يُعد كارثة أمنية تؤدي إلى اختراق كامل. 
cat hosts.txt | while read h; do
  for cfg in .env config.php database.yml settings.py parameters.yml web.config; do
    curl -sk --max-time 5 "${h}/${cfg}" | grep -m1 -iE "(password|secret|key|token)" && echo "[!] Found: ${h}/${cfg}"
  done
done | tee config-leaks.txt

grep -m1 -iE … يبحث عن كلمات سرية مثل password، secret، key، token.
&& echo … يُخرج رابط الملف عند العثور على سر.
| tee config-leaks.txt يحفظ النتائج في ملف نصي.
